# AI Mentor Playbooks

Use these templates when the user wants a concrete reusable method.

## 1. Clarify messy thoughts

Use when the user feels scattered, blocked, anxious, or unsure what the real task is.

Steps:

1. Ask for raw notes without requiring order.
2. Separate facts, assumptions, emotions, constraints, and decisions.
3. Identify the real goal behind the stated goal.
4. List unknowns that matter.
5. Produce the next small action.

Prompt:

```text
I am going to dump messy thoughts below. Do not solve immediately.
Please organize them into:
1. What I am trying to achieve
2. Facts
3. Assumptions
4. Emotions or worries
5. Constraints
6. Decisions I need to make
7. The next 15-minute action
```

## 2. Make a decision

Use when the user is comparing paths, tools, jobs, strategies, purchases, or priorities.

Steps:

1. Define the decision and deadline.
2. Identify criteria and weight them.
3. Compare 2-4 options.
4. Red-team the likely choice.
5. Produce a decision memo.

Prompt:

```text
Help me make this decision.
Decision:
Options:
Criteria:
Constraints:
Deadline:

Please produce:
1. A comparison table
2. The strongest argument for each option
3. The strongest argument against each option
4. What information would change the decision
5. Your recommendation, with confidence level
```

## 3. Plan a project

Use when the user wants to execute a project, launch something, organize work, or move from idea to plan.

Steps:

1. Define outcome and done criteria.
2. Break into phases.
3. Identify dependencies and risks.
4. Convert phases into tasks.
5. Choose the first milestone.

Prompt:

```text
Turn this project into an execution plan.
Goal:
Deadline:
People involved:
Resources:
Constraints:

Output:
1. Success definition
2. Work phases
3. Task list
4. Dependencies
5. Risks and mitigations
6. First milestone
7. What I should do today
```

## 4. Write or communicate

Use for emails, reports, proposals, hard conversations, performance reviews, summaries, and announcements.

Steps:

1. Identify audience and desired effect.
2. Gather facts and emotional tone.
3. Draft a clear version.
4. Create alternate tones if useful.
5. Review for ambiguity, blame, and missing asks.

Prompt:

```text
Help me communicate this clearly.
Audience:
Desired outcome:
Facts:
Tone:
Constraints:
Things to avoid:

Please produce:
1. Recommended message structure
2. Draft
3. A shorter version
4. Risks or phrases that may be misunderstood
```

## 5. Learn a new topic

Use when the user wants to learn efficiently, build a curriculum, or understand a difficult concept.

Steps:

1. Clarify target skill and current level.
2. Identify the smallest useful foundation.
3. Build a staged path.
4. Include practice tasks.
5. Add self-tests and feedback loops.

Prompt:

```text
Design a learning plan for me.
Topic:
Current level:
Goal:
Time available:
Preferred learning style:

Please output:
1. What to learn first
2. 2-week plan
3. Practice tasks
4. Common traps
5. Self-test questions
6. How to use AI as tutor, quizzer, and reviewer
```

## 6. Manage attention and overwhelm

Use when the user has many tasks, feels stuck, or needs prioritization.

Steps:

1. Collect all tasks.
2. Classify by impact, urgency, energy, and dependency.
3. Choose only the next 1-3 actions.
4. Shrink the first action to 10-15 minutes.
5. Remove or defer the rest.

Prompt:

```text
Help me triage my workload.
Here are all my tasks:

Please sort them by:
1. Importance
2. Urgency
3. Energy required
4. Dependencies
5. Can delegate / can automate / must do myself

Then tell me:
1. The top 3 for today
2. What to ignore for now
3. The first 10-minute action
```

## 7. Prepare or follow up on a meeting

Use for agendas, stakeholder alignment, meeting notes, decisions, and action items.

Steps:

1. Define meeting outcome.
2. Prepare agenda and decision questions.
3. Capture notes into decisions, open questions, and tasks.
4. Draft follow-up message.

Prompt:

```text
Help me prepare for this meeting.
Meeting topic:
Participants:
Desired outcome:
Context:
Sensitive issues:

Output:
1. Agenda
2. Questions to ask
3. Decisions needed
4. Risks to manage
5. Follow-up template
```

## 8. Reflect and turn experience into a reusable system

Use after a project, conversation, failure, success, or repeated task.

Steps:

1. Summarize what happened.
2. Identify what worked and failed.
3. Extract principles.
4. Convert into a checklist or prompt.
5. Define when to reuse it.

Prompt:

```text
Help me turn this experience into a reusable method.
What happened:
What worked:
What failed:
Constraints:

Please produce:
1. Lessons learned
2. Mistakes to avoid
3. A reusable checklist
4. A reusable AI prompt
5. When this method should be used again
```
